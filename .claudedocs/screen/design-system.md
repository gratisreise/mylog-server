# MyLog 디자인 시스템 및 공통 컴포넌트

`.claudedocs/screen/screens.md`의 화면 요구사항을 분석하여 도출한 디자인 시스템 및 공통 UI 컴포넌트 목록입니다. 이를 기반으로 일관성 있는 디자인과 재사용 가능한 컴포넌트를 구현할 수 있습니다.

---

## 1. Input Fields (입력 컴포넌트)

입력 필드는 상태(기본, 입력 중, 오류, 비활성)에 따른 일관된 스타일링과 유효성 검사 피드백이 필요합니다.

- **TextInputField (기본 텍스트 입력)**
  - **구성 속성:** 힌트 텍스트(placeholder), 에러 메시지, 최대/최소 글자 수(maxLength), 포커스 활성화 처리.
  - **사용처:** 이메일, 이름, 닉네임, 게시글 제목(5-30자), 카테고리 이름(1-12자) 등.
- **PasswordInputField (비밀번호 전용 입력)**
  - **구성 속성:** 입력 내용 숨기기/보이기 토글 아이콘 (눈 모양 아이콘), 비밀번호 양식 가이드.
  - **사용처:** 로그인, 회원가입, 프로필 수정 내 비밀번호.
- **TextAreaField (다중 라인 입력)**
  - **구성 속성:** 글자 수 카운터 표기 (예: 0 / 3000), 자동 높이 조절 또는 고정 스크롤 영역 적용.
  - **사용처:** 게시글 본문(10-3000자), 댓글 내용(5-200자), 프로필 소개글(최대 200자).
- **SearchBar (검색 필드)**
  - **구성 속성:** 검색 아이콘, 검색어 초기화 기능(`x` 버튼), `onSubmit` 액션.
  - **사용처:** 게시글 검색 필드.

## 2. Buttons & Controls (버튼 및 조작 컴포넌트)

- **PrimaryButton (주요 버튼)**
  - **특징:** 화면에서 가장 중요한 액션 강조 (브랜드 메인 컬러 반영).
  - **사용처:** 로그인, 회원가입, 작성, 수정, 생성 버튼 등.
- **SecondaryButton / OutlineButton (보조 버튼)**
  - **특징:** 취소, 뒤로가기 등 주요하지 않으나 필요한 동작 (배경이 없거나 테두리만 있는 형태).
  - **사용처:** 취소, 프로필/게시글 관리 등 보조 기능 버튼.
- **TextButton / LinkText (텍스트 링크)**
  - **특징:** 아이콘 또는 밑줄 등이 포함된 단순 텍스트 기반 링크.
  - **사용처:** 회원가입 링크, 이메일 로그인 링크 등 화면 이동용.
- **SocialLoginButton (소셜 로그인 버튼)**
  - **특징:** Google, Kakao, Naver 각 규정에 맞는 브랜드 컬러 및 전용 로고 위치 구성.
  - **사용처:** 소셜 로그인 화면.
- **ToggleSwitch (토글/스위치)**
  - **특징:** `On/Off` 상태의 직관적 노출 컴포넌트.
  - **사용처:** 알림 설정 화면(알림 활성화/비활성화).

---

## 3. List & Cards (리스트 및 카드 컴포넌트)

- **PostCard (게시글 카드)**
  - **구성 요소:** 
    - 썸네일 영역 (기본 이미지 폴백 제공)
    - 타이틀, 카테고리, 태그 리스트 컴포넌트
    - 작성자 메타데이터(프로필/이름), 작성일시 표시
  - **사용처:** 전체/내 게시글 목록, 검색 결과 목록 화면.
- **CommentItem (댓글 리스트 아이템)**
  - **구성 요소:** 
    - 작성자 이름, 아바타, 작성일시/수정일시 안내
    - 본문
    - 액션 조작부: [수정/삭제] (작성자 전용), [대댓글 달기] (공통)
    - 대댓글일 경우 들여쓰기(Padding 처리) 옵션
  - **사용처:** 게시글 상세 내 댓글.
- **NotificationItem (알림 아이템)**
  - **구성 요소:** 읽음 유무에 따른 글씨 굵기/배경색 차이 부여, 알림 타입 표시.
  - **사용처:** 알림 목록 화면.
- **CategoryListItem (카테고리 리스트 아이템)**
  - **구성 요소:** 카테고리명 표기 및 우측 끝 관리 아이콘 그룹(수정, 삭제).
  - **사용처:** 카테고리 목록 화면.

---

## 4. Selection & Filter (선택/필터 컴포넌트)

- **TagChip (기본 태그 칩)**
  - **구성 속성:** 텍스트(해시태그), 모서리가 둥근 컨테이너 형태.
  - **사용처:** 게시글 조회 상세 화면, 게시글 카드 내부에서의 목록 처리.
- **TagInputController (태그 입력기)**
  - **구성 속성:** 텍스트를 입력하고 Enter나 스페이스 입력 시 다중 칩으로 만들어짐(수정 시 제거 기능 포함).
  - **사용처:** 게시글 작성 및 수정, 검색의 태그 필터 영역.
- **CategorySelector (카테고리 선택기)**
  - **구성 속성:** Dropdown 혹은 BottomSheet 형태의 선택 항목 컴포넌트.
  - **사용처:** 게시글 작성 및 수정 화면의 속성 설정.

---

## 5. Media & Avatars (미디어 및 아바타 컴포넌트)

- **ProfileAvatar (프로필 아바타)**
  - **특징:** 원형으로 크로핑 적용 및 기본 이미지 폴백 제공. 필요에 따라 여러 크기(Small/Medium/Large) 베리에이션.
  - **사용처:** 내 프로필, 댓글 작성자 이미지, 게시글 상세 작성자.
- **ImagePickerUploader (이미지 업로더)**
  - **특징:** 이미지 첨부, 미리보기 썸네일 노출, 파일 삭제 기능 컨테이너 묶음 (`+` 버튼 등) 구성.
  - **사용처:** 게시글 내 이미지 첨부, 프로필 사진 업로드, 작성 시 썸네일 지정.

---

## 6. Feedback & Layout (피드백 및 레이아웃 컴포넌트)

- **Pagination & LoadMore (페이지네이션)**
  - **특징:** 명시적 페이지 번호 블록이거나 무한 스크롤 형태(Loading Spinner 제공)의 조작 부.
  - **사용처:** 리스트 형태의 데이터 처리 부분 전체 라우트(게시글, 검색 결과, 댓글 목록).
- **Dialog / BottomSheet (오버레이 알림창)**
  - **특징:** 경고 다이얼로그 (Warning/Destructive 시 붉은 폰트 컬러 등 반영) 혹은 선택 다이얼로그 노출.
  - **사용처:** 삭제 전 재확인(`정말 삭제하시겠습니까?`), 권한 경고 창 등.
- **Global Header (공통 상단 네비게이션 앱바)**
  - **특징:** 화면 타이틀, 좌측 뒤로 가기 액션, 우측 글로벌 버튼(검색, 프로필 아이콘 등)의 유연성 처리 지원.
  - **사용처:** 모든 서브 화면.
