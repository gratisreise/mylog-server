1. Starting Your Stitch Project

You can begin with a high-level concept or a detailed description.

High-level: Great for brainstorming or broad ideas.

Example: “An app for marathon runners.”

Detailed: Useful when you want specific UI elements or features up front.

Example: “An app for marathon runners to engage with a community, find partners, get training advice, and find races near them.”

Use adjectives to describe the vibe/mood—it influences colors, fonts, and layout decisions.

2. Refining Design Iteratively

Make small, targeted changes rather than huge overhauls.

Focus on one screen or component at a time and clearly say what to change and how.

Example: “On the homepage, add a search bar to the header.”

Example: “Change the primary call-to-action button on the login screen to be larger and use the brand’s primary blue color.”

Describe imagery clearly (e.g., album art, backgrounds) so Stitch can match your intent.

3. Controlling Theme and Style

Colors: Ask for specific color values or describe the mood you want (e.g., warm, cool).

Typography and Borders: Direct how fonts and UI elements should appear (serif vs sans, rounded buttons, etc.).

Example: “Use a playful sans-serif font.”

4. Modifying Images

Specify which images and what kind of change you want (backgrounds, icons, etc.).

If you’re changing theme colors, tell Stitch whether images should match the new palette.

5. Changing App Text Language

You can command Stitch to translate all on-screen text to another language:

Example: “Switch all product copy and button text to Spanish.”

Pro Tips for Better Results

Be clear and concise—avoid vague instructions.

Iterate & experiment—rephrase and try again if the result isn’t right.

One major change at a time improves predictability.

Use UI/UX terminology (e.g., navigation bar, card layout) to guide Stitch’s understanding.

Reference specific elements (like a button on a specific screen) rather than generic descriptions.
